# 视觉 SLAM 中基于汉字语义正交先验的单应性矩阵消歧义算法总结

## 1. 研究背景与核心挑战 (Background & Motivation)

在单目视觉 SLAM 或视觉里程计（VO）中，面对室内或城市场景中的平面目标（如路牌、招牌），通常利用单应性矩阵（Homography, $H$）来恢复相机的相对位姿。

然而，对 $H$ 矩阵进行数学分解（Decomposition）在物理上具有**多解性**：
* **手性校验（Chirality Check）**：通过要求物体必须在相机前方，通常能将 4 个数学解减少为 2 个。
* **二选一困境**：剩下的 2 个解中，一个是反映真实运动的“真实解”，另一个是数学上的“幽灵解”（假解）。传统方法依赖于大规模特征点的三角化验证，计算开销大，且在相机正对平面的退化场景下极易失效。

## 2. 核心创新点：3D 逆投影正交先验 (Semantic Orthogonality Prior)

针对传统方法的不足，本项目提出了一种利用汉字排版语义的创新校验算法。其核心物理假设为：**汉字的横向笔画与竖向笔画在 3D 物理世界中是严格垂直（正交）的。**

该方法无需复杂的三角化，而是将图像上的 2D 笔画特征通过候选解的几何参数**逆投影（Back-projection）**回到 3D 空间。通过测量笔画在 3D 空间中的垂直程度（点乘误差），实现对假解的瞬间击杀。

## 3. 算法管线与模块实现 (System Architecture)

实验由三个相互解耦的 Python 模块组成，实现了从像素提取到 3D 位姿判定的完整流程。

### 模块 A：文本骨架高精度提取 (`lines_tool.py`)
该模块负责在复杂的背景中压榨出最纯净的汉字几何特征：
* **图像增强**：利用双边滤波（Bilateral Filter）保留边缘的同时去除噪点，并结合锐化算子增强文字对比度。
* **激进线段检测**：使用 LSD (Line Segment Detector) 算法，通过高精度的直线拟合捕获汉字细节。
* **质心距离过滤（核心优化）**：计算所有线段的全局质心，基于 2$\sigma$ 标准差原则剔除远离文字区的边框干扰线及背景杂噪，仅保留核心文本骨架。

### 模块 B：基于深度学习的 ROI 匹配 (`Homography_loftr_tool.py`)
为 H 矩阵的解算提供高精度的点对支撑：
* **交互式多边形掩码**：允许用户自定义 ROI，隔绝无关区域（如行人、天空）的干扰。
* **LoFTR 深度匹配**：采用基于 Transformer 的无检测器匹配网络，在低纹理的路牌表面依然能获得密集的匹配点。
* **空间防抖过滤**：在鼠标交互中加入距离防抖逻辑，并结合 RANSAC 算法确保 $H$ 矩阵的地基稳固。

### 模块 C：3D 正交校验引擎与可视化 (`main.py`)
系统的大脑，负责逻辑判定与三维重建：
* **3D 空间正交误差评估**：遍历解空间，将横竖笔画射回 3D 空间，计算向量点乘。若点乘结果趋近于 0，则判定为物理真实平面。
* **SLAM 级 3D 可视化**：将相机运动轨迹、坐标系变换以及路牌平面的 3D 物理轮廓在 Matplotlib 三维空间中进行 1:1 的真实重现，支持鼠标交互旋转观察。

## 4. 实验结论 (Conclusion)

1. **鲁棒性提升**：该思路成功解决了单应性分解中“正视退化”导致的灭点计算崩溃问题。
2. **计算高效性**：逆投影正交校验属于线性矩阵运算，耗时远低于传统的非线性优化及多点三角化验证。
3. **工程价值**：在城市辅助驾驶、室内机器人寻路等特定场景中，该语义先验可作为极其可靠的位姿消歧义工具。
